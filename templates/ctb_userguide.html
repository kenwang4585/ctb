General introductions

About the input data to run the tool?

  • 3a4 for required DF ORG (without option, download via UOV 3a4 view AutoCTB)
  • Supply data downloaded from Kinaxis (Contact SME Jully zhou if any question about how to download)
  • TAN supply from PCBA allocation file
    • Important note#1: Pls try to ensure consistent timing for getting above data!
    Important note#2: the CTB is based on these TAN/CPN supply data provided - ensure your supply data include all TAN/CPN you want to use!

    What is Exceptional priority?

    PSP can give an exceptional priority to a non-priority SS so they can also get an absolute high priority despite OSSD and reflect in allocation - this should be used only exceptionally. Default priority ranking#: PR1:1; PR2:2; PR3:3; L4:4; BUP:5. PSP can set exceptional priority via the link you can find in the footer. E.g., if one SS is added with Ranking# 4.1, this SS will be ranked after L4; or if added with Ranking# greater than 5, they will be after BUP; etc.

    Other exceptions considered

    GIMS, Config issue, etc., data captured from CM CTB stored in the CM_CTB smartsheet.

    Additional columns created in 3a4 and interpretations

    distinct_po_filter

    Duplicated rows are introduced when we explode the BOM. Filter "YES" to show just one line for one PO. This is useful especially when you simply want to sum up the "C_UNSTAGED_DOLLARS" and this will avoid duplication.

    BOM_PN

    This is generated based on "FLB_TAN".

    EXCEPTION_NAME

    This is auto-captured from CM CTB file in smartsheet for related PO to indicate if one PO has any exceptions issues like GIMS, Config issue, etc., which prevents build.

    priority_cat

    The priorities categories - PR1, PR2, PR3, L4, BUP - are collected and indicated here in this column. DPAS_RATING is also referenced to ensure order correctly categorized as PR1.

    priority_rank

    This column has an integer or float number to indicate the priority ranking# for above priority categories; if there is exceptional priority input, that new exceptional ranking# is shown here instead.

    ss_overall_rank

    This is the final ranking for each SS based on the ranking logic we used.

    min_date

    This is a min date of ORIGINAL_FCD_NBD_DATE and LT_TARGET_FCD; for Non_REVENUE orders it's same as CURRENT_FCD_NBD_DATE. If the dates are blank, then consider it as 30days from now.

    earliest_allowed_pack_date

    Default based on LT_TARGET_FCD - 30days; for MFG_HOLD or Unscheduled PO, this date must not be within 30days from today; if there are other EXCEPTIONS (Indicated in Col EXCEPTION_NAME), then this date must not be within 14days from today. If order is scheduled but LT_TARGET_FCD is missing, then it follows CURRENT_FCD_NBD_DATE - 30days.

    po_ctb

    This is generated based on "po_supply_ready_date", "earliest_packable_date" and FLT value.

    ss_ctb

    This is based on the latest po_ctb date within one SS.

    ss_updated_status

    This indicates "ss_ctb" vs "CURRENT_FCD_NBD_DATE".

    RISO (as is)/RISO (to be)

    These 2 col indicate based on LT target what's the current status based on FCD, and what's the new status based on "ss_ctb".

    Summaries

    build_projection

    Summary based on po_ctb (for date columns); aggregated into weekly buckets with Sunday as start date.

    agg_fcd_vs_ctb

    Summary based on current FCD and po_ctb to indicate pull in opportunities (positive cum-delta) or risks (negative cum-delta). Columns are aggregated into weekly buckets with Sunday as start date.

    build_impact_wk0,build_impact_qend

    Gating items summary from current week or Qend standpoint.
    Pack cutoff is Saturday of the week; Supply cutoff is Thursday of the week. "OUTSIDE_WINDOW" is defined based on pack cutoff date + 30 days, e.g., for wk0, it means end of current week + 30days, for Qend, it means Qend + 30days.
